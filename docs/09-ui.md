# 09. UI設計

SwiftUIによるmacOSネイティブUI。3ペイン構成を基本とし、検索・取り込み・セットアップの各フローを定義する。

## 1. 画面構成: 3ペイン

```
+------------------+-------------------------------+----------------------------+
| サイドバー        | 論文リスト                     | 詳細ペイン                  |
|                  |                               |                            |
| 検索 [統合検索▾]  | タイトル | 著者 | 年 | venue  | [情報|PDF|ノート|引用グラフ] |
|------------------|-------------------------------|                            |
| ▼ ライブラリ      | Attention Is All You Need     |  Attention Is All You Need |
|   すべて          |   Vaswani+ 2017 NeurIPS ●     |  Vaswani, Shazeer, ...     |
|   最近追加        | BERT: Pre-training of ...     |  NeurIPS 2017              |
|   未読            |   Devlin+  2019 NAACL   ●     |                            |
|   処理中 (2)      | Scaling Laws for ...          |  abstract ...              |
| ▼ コレクション    |   Kaplan+  2020 arXiv   ◐     |                            |
|   ▼ Transformers |                               |  [BibTeXをコピー]           |
|     Efficiency   |                               |                            |
|   Survey         |                               |                            |
+------------------+-------------------------------+----------------------------+
| ＋ 取り込み        ステータスバー: ジョブ進捗 [convert 2/3 ▸]                    |
+------------------------------------------------------------------------------+
```

`NavigationSplitView`（3カラム）で実装。右ペインは折りたたみ可能。

## 2. サイドバー

### 2.1 スマートリスト

`papers.status`（→ [02](02-data-model.md)）に基づく固定リスト。`is_stub = 1` の行はすべてのリストから除外する。

| リスト | 条件 |
|---|---|
| すべて | 全論文 |
| **お気に入り** | `is_favorite = 1`（→ [02](02-data-model.md)） |
| **自著論文** | `is_own = 1`。選択直後（論文未選択時）は詳細ペインに**自著被引用ネットワーク**を表示（→ 4.1節） |
| 処理中 | `status IN ('converting')` または実行中ジョブあり。件数バッジ表示 |
| **PDF未取得** | `status = 'metadata_only'`。paywall等でPDFが取れなかった/書誌のみ登録した論文。書誌・BibTeX・タイトル+アブスト検索は機能している（部分的成功であり緊急性は低い）。件数バッジ表示 |
| **書誌未解決** | `status = 'pdf_only'`。手動解決が必要な真の要対応。件数バッジ表示 |
| **失敗** | `status = 'failed'`。リトライ/削除の判断待ち。件数バッジ表示 |

> **設計変更（2026-06）**: 当初の「要対応」（3状態の合算リスト）は範囲が広く曖昧だったため、
> 性質別の3リストに分割した。また「最近追加」はソート機能（追加日降順が既定）と重複するため廃止した。とくにpaywall由来の `metadata_only` は件数が多いのに緊急性が低く、
> 「失敗」と同じリストに混ざるのが混乱の原因だった。

### 2.2 お気に入り・自著論文の登録

- 論文リスト行のコンテキストメニュー、および情報タブのトグルボタン（★ / 人物アイコン）で付け外しする
- フラグの正本は `meta.json`（→ [03](03-library-layout.md)）。インデックス再構築で復元される
- 旧コレクション機能は廃止（→ [02](02-data-model.md) の設計変更メモ）

## 3. 論文リスト

| カラム | 内容 |
|---|---|
| タイトル | `papers.title`。2行目に著者（`First+` 形式の省略表記） |
| 年 / venue | `year` / `venue` |
| status | バッジ: `indexed` ●（緑）/ `converting` ◐（スピナー）/ `metadata_only`・`pdf_only` ▲（黄）/ `failed` ✕（赤）。`conversion_warnings > 0` の行は文字化け警告アイコンを併記（→ [05](05-pdf-conversion.md) 4.1節） |

- **選択のライフサイクル**: サイドバーのリスト切替時は**論文選択を常に解除**する
  （切替＝コンテキストの切替として挙動を予測可能にし、「自著論文」では着地時に必ず
  被引用ネットワーク（4.1節）が表示されるようにするため）。**⎋（ESC）キーでも選択を解除**できる
  （リストが長く空白クリックできない場合の標準経路）
- ソート: 追加日（既定・降順）/ 年 / タイトル / 第一著者
- 絞り込み: 年範囲、status、venue のトークンフィルタ（リスト上部）
- **削除**: 3経路を提供する — ①行のコンテキストメニュー「削除…」、②選択行への **⌫（Delete）キー**、
  ③リストペインの**ツールバーのゴミ箱アイコン**（選択中のみ有効、⌘⌫ショートカット併設）。
  いずれも破壊的操作のため**確認ダイアログ**（タイトル表示 + 「ゴミ箱に移動」）を経て実行する。
  実体はディレクトリごとゴミ箱へ移動（→ [03](03-library-layout.md) 6節）
- **ステータス別リストの一括削除**: 「PDF未取得」「書誌未解決」「失敗」の各リスト選択中は
  ツールバーに「**このリストをすべて削除…**」を表示し、リスト内の全論文を件数つきの
  確認ダイアログを経て一括でゴミ箱へ移動する（例: paywallで取れなかった論文群の一掃）
- `pdf_only` の行は「メタデータ未解決」バッジから手動解決UI（DOI/arXiv ID入力、Crossref候補リストからの選択 → [04](04-ingest-pipeline.md) 4節）を開ける

## 4. 詳細ペイン

タブ構成。

| タブ | 内容 |
|---|---|
| 情報 | メタデータ（編集可能フィールド）+ abstract + 「BibTeXをコピー」ボタン（→ [02](02-data-model.md) 2節）+ **「引用をコピー」メニュー**（APA / MLA / Chicago / IEEE / Vancouver からスタイルを選んで整形済み引用文をコピー → [02](02-data-model.md) 2.4節。最後に使ったスタイルを記憶）+ **「Webページを開く」ボタン**（論文のWebページをブラウザで開く。リンク先の優先順: DOI（`https://doi.org/{doi}`、出版社ページへ解決）→ arXiv abstページ → `papers.url`。いずれも無い論文では非表示）+ **「PDFを書き出し」ボタン**（ライブラリ内部の `paper.pdf` を、人に送れる体裁のファイル名で任意の場所へコピーする。ファイル名は `{第一著者の姓}[ and {第二著者の姓} \| et al.] {年} - {タイトル}.pdf`。ファイル名に使えない文字は除去し、長すぎるタイトルは切り詰める。PDF未取得の論文では非表示） |
| PDF | PDFKitビューア（→ 5節）。下部に**「添付ファイル」セクション**: `supplements/` の一覧（クリックで既定アプリで開く）、＋ボタン/ドロップで追加（任意のファイル種別・複数可）、コンテキストメニューで削除（ゴミ箱へ移動、復元可能 → [03](03-library-layout.md)）。`paper.pdf` がない場合は取得状況と**ドロップ領域**、および**「代替PDF（プレプリント等）を自動で探す」ボタン**（→ [04](04-ingest-pipeline.md) 6節）を表示。ここへのドロップは**この論文へのPDF添付**として扱われ（推測なしの確実な合流 → [04](04-ingest-pipeline.md) 4節）、pdf_hash重複チェックの上でconvert以降が再開される（→ [04](04-ingest-pipeline.md) 6節） |
| Markdown | **有効Markdown（`paper.corrected.md` 優先、なければ `paper.md` → [05](05-pdf-conversion.md) 5.2節）の閲覧**。AI（MCP `get_fulltext` / RAG）が読むのはこのMarkdownであり、変換ミスがあるとユーザが気づかないままAIが誤った本文を参照し続けるリスクがある。PDFタブと見比べて変換品質を確認する手段として提供する。レンダリングは簡易（見出し・段落・表・リスト・コードが読める水準）でよく、リッチである必要はない。数式はLaTeX文字列のまま表示、`<!-- image -->` はプレースホルダ表示。ヘッダに**変換品質警告バッジ**（`conversion_warnings` → [05](05-pdf-conversion.md) 4.1節）・**「高精度で再変換」ボタン**（reconvertジョブ投入 → [05](05-pdf-conversion.md) 5.1節）・修正版表示中のインジケータを置く |
| ノート | Markdownエディタ。正本は `papers/{uuid}/notes.md`、保存時に `notes` 索引を更新 |
| 引用グラフ | エゴネットワーク表示（→ [08](08-citation-graph.md)） |

### 4.1 自著被引用ネットワーク（自著論文リスト選択時）

サイドバーで「自著論文」を選択した直後（**論文未選択の状態**）に、詳細ペインへ
**登録した自著論文を中心とした被引用ネットワーク**を表示する。

> 位置づけ: 検索や管理の利便性ではなく、**ユーザに満足感・達成感を与えるための機能**。
> 表示は他画面より意図的にリッチに作る。

| 要素 | 表現 |
|---|---|
| 統計ヘッダ | 「自著 N本 / 総被引用 M件 / 引用元（ユニーク）K件」を大きなタイポグラフィで表示 |
| 背景 | ダークグラデーション（発光感のあるノードを際立たせる） |
| 自著ノード | 金色（ラジアルグラデーション + グロー）。サイズは被引用数の対数スケール |
| 引用元ノード | 出版年で色温度を変える（新しい引用ほど明るい = 「影響が広がっている」感覚） |
| エッジ | 引用元 → 自著。半透明の発光ライン |
| レイアウト | force-directed（→ [08](08-citation-graph.md)）を**収束までアニメーション表示** |
| クリック | 自著ノード→その論文を開く。stubノード→詳細パネル（取り込みボタン付き） |

データは `citations` キャッシュから構築する（自著集合への入エッジ + 自著間の引用エッジ）。
被引用が未取得の自著があれば、表示時にバックグラウンドで取得を投入する（→ [08](08-citation-graph.md) 2節）。

### 4.2 ライブラリ概況（論文未選択時）

論文を選択していないとき、詳細ペインには空のプレースホルダではなく**選択中リストの概況**を表示する
（「次に何をするか」を示す場所。自著論文リストのみ4.1節のネットワークが出る）。

**通常リスト（すべて / お気に入り / 処理中）**: ダッシュボード

| 要素 | 内容 |
|---|---|
| 統計 | 論文数 / 全文検索可能（indexed）数 / ノートあり数を大きな数字で |
| 年代分布 | 出版年のミニヒストグラム（Canvas） |
| 最近追加 | リスト内の直近5件。クリックでその論文を選択（廃止した「最近追加」リストの受け皿） |
| 対応待ちサマリ | PDF未取得 / 書誌未解決 / 失敗の件数。クリックでそのリストへ移動 |
| 品質警告 | 文字化けの疑いがある論文数（高精度再変換の案内） |
| ヒント | 自然言語検索・MCP連携の一行Tips |

**処理中リスト**: ジョブモニタ

- 実行中ジョブ: 論文タイトル + **ステージのプログレス表示**（resolve → fetch → convert → chunk → embed → index、
  完了済み/実行中/未到達を色分け）+ 経過時間
- 待機中ジョブの一覧、失敗ジョブ（再試行 / 無視ボタン付き）、ワーカー状態
- ステータスバーのポップオーバー（一時表示）と違い、一括取り込みの進捗を腰を据えて眺める場所。
  キューが空のときは「処理中のジョブはありません」

**ステータス別リスト（PDF未取得 / 書誌未解決 / 失敗）**: 説明 + アクションパネル

- そのリストの意味の説明（例:「paywall等でPDFを取得できなかった論文です。書誌・BibTeX・
  タイトル+アブスト検索は機能しています」）と対処方法の案内
- 主要アクション（一括削除等）。リストが空のときはその旨を表示

## 5. PDFビューア

- `PDFKit` の `PDFView` を `NSViewRepresentable` でラップ。ページサムネイル（`PDFThumbnailView`）と文書内テキスト検索を提供
- **検索ヒットからのページジャンプ**: 検索結果はチャンク単位（`chunks.section_path`）だが、チャンクはMarkdown由来でPDFページ番号を直接持たない。v1では近似対応とする:
  1. `paper.docling.json` の各要素が持つ **provenance（ページ番号）** を参照し、チャンク先頭要素の所属ページを引く
  2. `PDFView.go(to:)` で該当ページへジャンプし、チャンク先頭文をページ内 `PDFDocument.findString` でハイライト（見つからなければページ移動のみ）
- 精密なバウンディングボックス対応（座標ハイライト）はv2課題（→ [10](10-roadmap-risks.md)）

## 6. 検索UI

- ツールバーの**統合検索フィールド**（⌘F）。入力確定で検索モードに遷移し、中ペインが検索結果リストに切り替わる
- モード切替は**検索スコープバー**（macOS標準UI。論文リスト上部に検索アクティブ時のみ表示される）。
  検索結果表示中にモードを切り替えると即座に再検索する:

| モード | 動作 |
|---|---|
| ハイブリッド（既定） | FTS5 + semantic をRRFで統合（→ [06](06-search-rag.md)） |
| キーワードのみ | FTS5のみ。ワーカー未起動でも即応答（embedding不要） |

- 検索モードの終了: 検索語を空にする、または**サイドバーのリストを切り替える**と通常のリスト表示へ戻る
  （リスト切替＝コンテキスト切替の原則。3節の選択解除と同じ規則）
- 結果リストは**論文単位にグルーピング**し、論文ごとにヒットチャンクの snippet と `section_path`（例: `3. Method > 3.2 Training`）を最大3件表示
- **ヒット強度の表示**:
  - 論文行: トップヒットを100%とした**相対強度バー**（統合スコアはRRF=順位ベースで絶対値に意味がないため、相対表示とする）
  - チャンク行: `match_type` の色チップ（hybrid / semantic / keyword）+ **semanticコサイン類似度**（「意味 72%」。正規化ベクトルのため0〜1で解釈可能）+ **キーワード順位**（「キーワード #2」。BM25生値は非有界で解釈しづらいため順位表示）
  - snippet中の**クエリ語をハイライト**（keyword / hybridヒットのみ。semantic一致には語の対応が存在しないため対象外）
- 結果クリックで詳細ペインを開く。チャンクヒットのクリックは**Markdownタブの該当セクションへスクロール**する（v1）。PDFタブの該当ページへのジャンプ（provenance近似 → 5節）はv1.x課題
- semantic検索の初回はワーカー起動待ち（数秒）が発生するため、フィールド下に「モデル準備中…」を表示する

## 7. 取り込みUI

すべての経路が `jobs` キューに合流する（→ [04](04-ingest-pipeline.md)）。

| 経路 | UI |
|---|---|
| **「＋ 取り込み」ボタン（サイドバー下部に常設）** | 1節モックアップどおりサイドバー最下部に固定表示する（ツールバー置きだとサイドバーが狭いときオーバーフロー「»」に畳まれてしまうため）。**2ボタン構成**: ①主ボタン「＋ 取り込み」（全幅・borderedProminent）→ 入力ダイアログ（⌘N） ②フォルダアイコンの副ボタン → シートを経由せず直接ファイル選択ダイアログ（⌘O）。Menuのラベルはカスタム描画・ホバーが効かない（AppKitポップアップセル描画）ため使わない。ショートカットはメニューバー「ファイル」にも登録し、常時有効 |
| 入力ダイアログ | 入力種別は自動判別。**クリップボードにURL / DOI / arXiv IDらしき文字列があれば自動プリフィル**し（判定は `PaperIdentifier.parse`）、「クリップボードから入力しました」と明示する。コピー → ⌘N → Enter の実質2操作で取り込み完了。PDFはウィンドウへのドロップでも取り込める旨のヒントを一行表示 |
| ファイル選択ダイアログ | PDFファイル（複数選択可）またはフォルダを選択。**フォルダ指定時は再帰的に走査し、中のPDFを一括投入**する。件数が多い場合（5件以上）は確認ダイアログを経る（誤って巨大フォルダを選んだ際の大量ジョブ投入防止） |
| PDFドラッグ&ドロップ | **ウィンドウ全体**をドロップターゲットとし、オーバーレイ表示。複数ファイル可。**フォルダのドロップも同様に再帰走査して一括投入** |
| `paperd://import` URLスキーム | 外部ツール連携用の汎用入口（→ [01](01-architecture.md) 6節）。確認ポップアップを経て取り込み |

一括投入された各PDFは通常の取り込みパイプライン（→ [04](04-ingest-pipeline.md)）を個別ジョブとして流れる。
重複は pdf_hash / 解決後DOI で検出され、既存エントリへの合流・cancelで処理される（→ [04](04-ingest-pipeline.md) 4・5節）。

### 7.1 ジョブ進捗（jobsテーブルのUI）

- ステータスバーに実行中ジョブ数とステージを集約表示。クリックでジョブ一覧ポップオーバーを開く
- ステータスバー右端に**ワーカーインジケータ**（● 稼働中(バージョン) / ○ 停止 / △ 未セットアップ）を常時表示。
  **どの状態でもクリックで設定のワーカータブを開く**（MCPインジケータと統一。起動/停止の操作は設定タブ側）。
  未セットアップ時はリスト上部に**セットアップ誘導バナー**も表示し、設定画面へ導く
  （初回ウィザード（8節）の前段。完了まで毎回表示される）
- その隣に**MCPインジケータ**を常時表示: 最終アクセス（→ [07](07-mcp-server.md) 6節）があれば
  「MCP: ◯分前」、なければ「MCP: 未接続」（薄色）。**クリックで設定画面の連携タブを直接開く**
  （登録コマンドのコピー・最終アクセス詳細への導線）
- ジョブ一覧の各行: 論文タイトル（resolve前は入力文字列）/ 現在ステージ（`resolve → fetch → convert → chunk → embed → index`）のプログレス / 経過時間
- `failed` ジョブは `last_error` と**再試行ボタン**を表示（失敗ステージから再開 → [04](04-ingest-pipeline.md) 7節）
- 失敗ジョブの**無視（クリア）**: ジョブ一覧の各行の「無視」と、一覧上部の「**失敗をすべて無視**」で
  失敗ジョブの行を削除し、ステータスバーの失敗表示をリフレッシュする。再試行しないと判断した失敗を
  視界から消すための機能（論文行自体は消えない。`papers.status = failed` の行は要対応リストから削除できる）
- 重複検出（`cancelled`）はマージ提案ダイアログを表示（→ [04](04-ingest-pipeline.md) 5節）

## 8. 初回セットアップウィザード

`venv + pip install` によるPython環境構築（→ [01](01-architecture.md) 3.3節）をガイドする。完了まで取り込み機能は無効（閲覧系は利用可）。

```
[1 ライブラリ場所] → [2 Python環境構築] → [3 embeddingモデルDL] → [4 完了]
```

| ステップ | 内容 |
|---|---|
| 1. ライブラリ場所 | 既定 `~/PaperdLibrary`。変更可。既存ライブラリ検出時は「開く」を提案 |
| 2. Python環境構築 | `python -m venv .venv` → `pip install --upgrade -e .[ml]` の進捗（ダウンロード量・パッケージ数）を表示。**中断可・再開可**（pipのキャッシュにより再実行は差分のみ） |
| 3. モデルDL | Qwen3-Embedding-0.6B 4bit（約335MB）をHugging Face cacheへダウンロード。進捗バー + 残量表示 |

## 10. ローカリゼーション

国際配布（Homebrew）に向け、**レイヤごとに言語戦略を分ける**。一律のローカライズはしない。

| レイヤ | 言語方針 | 理由 |
|---|---|---|
| アプリUI（GUI文言） | **ja / en ローカライズ**（システム言語追従） | 唯一の人間向けGUI |
| PaperdCore のエラー・診断文 | **英語固定**（ローカライズしない） | `jobs.last_error` 等としてDBに**保存される**＝表示時でなく発生時の言語で固定されるため、ローカライズ機構と相性が悪い。診断情報として英語に統一 |
| MCPツール定義・応答 | **英語固定** | 読み手はAI。単一言語でよく、国際ユーザにも自然 |
| CLI（paperd-cli / paperd-mcp） | **英語固定** | 開発・ヘッドレス用途 |
| Claudeスキル（SKILL.md） | **英語固定** | Claudeは英語スキルでもユーザの言語で応答できる |

### 10.1 UIローカライズの機構

- **String Catalog**（`Sources/Paperd/Localizable.xcstrings`）+ `Package.swift` の `defaultLocalization: "ja"`
- **ベース言語は日本語**: SwiftUIの `Text("取り込み")` 等は文字列リテラルがそのままローカライズキーになる
  （ja はキー自身へフォールバックするため ja テーブル不要、xcstrings には **en 訳のみ**を記載する）。
  既存コードの書き換えを最小化するための選択
- 文字列補間を含むSwiftUIリテラルは自動でフォーマット指定子つきキー（例: `"%lld 件"`）になる
- 非SwiftUI経路の文字列（アラート文・`String` 変数として組み立てる文言）は `String(localized:)` でラップする
- **アプリバンドルでの解決**: `swift build`（CLI）は xcstrings を素通しコピーするだけで
  コンパイルしないため、`scripts/make-app.sh` が `xcrun xcstringstool compile` で
  `en.lproj/Localizable.strings` を生成し `Paperd.app/Contents/Resources/` 直下へ置く
  （`Bundle.main` 解決。ビュー側で `bundle:` 指定が不要になる。ja はキー自身への
  フォールバックなので ja.lproj は生成されない）。Info.plist に `CFBundleDevelopmentRegion: ja` と
  `CFBundleLocalizations: [ja, en]` を設定する（→ [01](01-architecture.md) 7節）。
  `swift run` の非バンドル実行ではローカライズされず常に日本語になる（開発用途のため許容）
- 検証は `defaults write app.paperd AppleLanguages '(en)'` での言語強制起動による目視確認
